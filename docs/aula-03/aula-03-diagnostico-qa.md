1. Diagnóstico da Situação Atual
1.1 Papéis atuais identificados

Liste quais papéis vocês acreditam que existem atualmente na startup.

Desenvolvedores
Analista de Sistemas
Gerente de Produto
1.2 Quem é responsável pela qualidade hoje?

Possivelmente não há um responsável específico pelo QA, sobrando para os desenvolvedores.

1.3 Problemas identificados

Liste os principais problemas relacionados à falta de organização da qualidade.

Ausência da definição clara de responsabilidades relacionadas à qualidade de software
Ausência de processos de teste antes da entrega
Ausência de registro e controle de bugs
1.4 Impactos desses problemas

Falhas no sistema, erros em pedidos ou entregas, insatisfação e irritação dos usuários.

1.5 A qualidade é responsabilidade de quem?

A qualidade de software deve ser entendida como uma responsabilidade compartilhada pela equipe, mesmo quando existe um papel específico de QA.

2.2 Descrição dos papéis

Preencha a tabela abaixo:

3. Práticas de QA Sugeridas

Sugira práticas que a startup pode adotar para melhorar a qualidade.

3.1 Lista de práticas
Revisão antes do deploy
Testes exploratórios
Registro e acompanhamento de bugs
3.2 Explicação das práticas

Explique brevemente cada prática sugerida.

Prática 1:

Descrição: Realizar testes manuais nos principais fluxos do sistema, como finalização de pedidos, garantindo que funcionem corretamente antes da entrega.

Prática 2:

Descrição: Executar testes exploratórios para encontrar falhas não previstas nos cenários tradicionais de teste.

Prática 3:

Descrição: Registrar todos os bugs encontrados em uma ferramenta (como GitHub Issues), permitindo acompanhamento e priorização.

5. Conclusão da Equipe

Descreva brevemente:

O que a equipe aprendeu com a atividade
Principais dificuldades encontradas
Principais melhorias propostas para a startup

Resposta: Com a atividade proposta, compreendemos a importância de uma estrutura organizada de qualidade dentro do desenvolvimento de software. Foi possível perceber que a ausência de papéis definidos e práticas de QA impacta diretamente na experiência do usuário.
A principal dificuldade foi estruturar claramente as responsabilidades de cada papel.
Como melhoria, propomos a definição de papéis, adoção de práticas de teste e a criação de uma cultura de responsabilidade compartilhada pela qualidade.

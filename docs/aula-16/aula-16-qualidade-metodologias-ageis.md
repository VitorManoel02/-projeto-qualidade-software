# 1. Análise de Práticas Ágeis no Processo

| Prática                        | Existe no processo? | Como é aplicada atualmente?                                                                              | Pode ser melhorada?                                                          |
| ------------------------------ | ------------------- | -------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| Planejamento iterativo         | Sim                 | As funcionalidades são desenvolvidas em etapas durante as aulas e entregas parciais do projeto.          | Sim, com planejamento mais detalhado por sprint.                             |
| Priorização de funcionalidades | Sim                 | As funcionalidades principais são implementadas antes das funcionalidades complementares.                | Sim, utilizando backlog priorizado por valor de negócio.                     |
| Entregas incrementais          | Sim                 | O sistema é desenvolvido gradualmente, com novas funcionalidades sendo adicionadas ao longo do semestre. | Sim, realizando entregas menores e mais frequentes.                          |
| Feedback frequente             | Parcialmente        | O feedback ocorre principalmente através do professor e dos testes realizados pela equipe.               | Sim, aumentando a frequência das revisões e validações.                      |
| Trabalho colaborativo          | Sim                 | Os integrantes dividem tarefas e compartilham informações durante o desenvolvimento.                     | Sim, utilizando mais práticas de revisão de código e programação em pares.   |
| Controle visual das atividades | Parcialmente        | O acompanhamento ocorre por conversas e organização informal das tarefas.                                | Sim, utilizando um quadro Kanban para visualizar o andamento das atividades. |
| Melhoria contínua              | Parcialmente        | Ajustes são realizados quando problemas são identificados durante o desenvolvimento.                     | Sim, realizando reuniões periódicas para retrospectiva do projeto.           |

### Conclusão

O processo de desenvolvimento do LocalEats já apresenta diversas características das metodologias ágeis, como desenvolvimento incremental, colaboração entre os integrantes e priorização de funcionalidades. Entretanto, algumas práticas ainda podem ser fortalecidas para aumentar a organização e a qualidade das entregas. A adoção de ferramentas visuais para controle das tarefas, reuniões de retrospectiva e critérios formais para início e conclusão das atividades pode contribuir para um processo mais eficiente. Além disso, um acompanhamento mais frequente do progresso e feedback contínuo ajudaria a reduzir retrabalho e melhorar a comunicação da equipe.

# 3. Definition of Ready (DoR)

Uma funcionalidade estará pronta para desenvolvimento quando:

1. O requisito estiver claramente descrito.
2. Os critérios de aceitação estiverem definidos.
3. As regras de negócio estiverem documentadas.
4. Não houver dúvidas pendentes por parte da equipe.
5. A funcionalidade estiver priorizada no backlog.
6. As dependências necessárias estiverem identificadas.
7. O esforço estimado para implementação tiver sido discutido pela equipe.

---

# 4. Definition of Done (DoD)

Uma funcionalidade será considerada concluída quando:

1. Todos os critérios de aceitação forem atendidos.
2. O código estiver implementado e funcionando corretamente.
3. Os testes realizados pela equipe forem aprovados.
4. Não existirem erros críticos conhecidos relacionados à funcionalidade.
5. A funcionalidade estiver integrada ao sistema principal.
6. A documentação necessária estiver atualizada.
7. A funcionalidade tiver sido validada pelos integrantes da equipe.
